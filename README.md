# OCR + LLM — распознавание и очистка текста при помощи локальной нейронной модели

Пайплайн для пакетной обработки скриншотов: распознаёт текст через PaddleOCR, очищает его от артефактов и мусора с помощью локальной LLM (Ollama), извлекает вопросы и варианты ответов, сохраняет результаты в Excel.

---

## Как работает

1. **Поиск данных** — скрипт ищет уже готовые JSON-файлы с результатами OCR. Если их нет — запускает PaddleOCR на всех изображениях в указанной папке.
2. **Извлечение текста** — из каждого JSON извлекается сырой текст и сохраняется в `raw_texts/`.
3. **Очистка через LLM** — текст отправляется в Ollama с подробным промптом: модель убирает мусор (метки вариантов, элементы интерфейса, OCR-артефакты), исправляет латиницу, похожую на кириллицу, и возвращает структурированный результат.
4. **Парсинг** — очищенный ответ разбивается на вопрос и список вариантов ответов.
5. **Сохранение** — очищенные тексты сохраняются в `cleaned_texts/`, итоговые данные по всем вопросам — в Excel-файл с автоподбором ширины колонок.

---

## Стек

| Компонент | Назначение |
|---|---|
| [PaddleOCR](https://github.com/PaddlePaddle/PaddleOCR) | Распознавание текста на изображениях |
| [Ollama](https://ollama.com/) + qwen2.5:7b | Локальная LLM для очистки OCR-текста |
| pandas + openpyxl | Формирование и сохранение Excel-отчёта |
| Python stdlib (re, glob, json, os) | Парсинг, работа с файлами |

---

## Требования

- Python 3.9+
- [Ollama](https://ollama.com/) — локальный сервер LLM
- Модель `qwen2.5:7b` (или другая, совместимая с Ollama)

Установка зависимостей:
```bash
pip install paddleocr pandas openpyxl requests
```

Загрузка модели:
```bash
ollama pull qwen2.5:7b
```

---

## Использование

1. Запустите сервер Ollama:
```bash
ollama serve
```

2. В файле `recognition_and_cleaning.py` укажите пути к папкам:
```python
input_folder = r"путь\к\скриншотам"
output_folder = r"путь\к\папке\результатов"
```

3. Запустите скрипт:
```bash
python recognition_and_cleaning.py
```

---

## Структура результатов
```
output_folder/
├── raw_texts/          # Сырой текст, извлечённый из OCR JSON
├── cleaned_texts/      # Очищенный текст после обработки LLM
└── excel_reports/      # Excel-файлы с вопросами и вариантами ответов
```

В Excel каждая строка — один вопрос. Колонки: `source_file`, `question`, `option_1`, `option_2`, ... (число колонок определяется автоматически).
